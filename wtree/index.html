Git Working Trees

A chamada Working Tree (ou Working Directory) é nada menos do que o diretório dos arquivos que você está atualmente trabalhando em. Para entender bem isso, é útil também entender ainda mais dois conceitos: staging area e o repositorio em si.

O Repositorio (repo) é o container que salva todas as mudanças já feitas nos arquivos do seu projeto (vide git log para ver como acessar o histórico de mudanças.)

A Stagin Area é onde commits são "preparados", onde acontece a comparação entre os arquivos na sua Working Tree e os do repo.

Estados de um arquivo

Um arquivo no git pode estar em três estados:

  • Modified: Mudança apenas na Working Tree
  • Staged: Arquivos foram adicionados e estão na Staging Area
  • Commited: Mudanças salvas no Repositorio
